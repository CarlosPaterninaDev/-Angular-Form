ngModel
Directiva necesaria:
<form #form="ngForm">

Modulo: FormsModule

Descripción: Se coloca la directiva en cada campo que queremos que Angular conozca, nos ayuda a construir el modelo del objeto del formulario. Se asocia con el atributo "name" de la entrada de datos

Esta directiva es muy beneficiosa si no se quiere rellenar el formulario al momento de cargar, de tal forma que la directiva ngModel trabaja de forma unidireccional en el template

Tip: Recopilación de información por primera vez

Go!

ng-untouched
<.ng-touched>

Modulo: FormsModule

Valor inicial: ng-untouched

Cuando el control pierde el focus se agrega de forma instantea la clase a ng-touched y se elimina la clase ng-untouched

Go!
Validation Class
ng-pristine
<.ng-dirty>

Modulo: FormsModule

Valor inicial: ng-pristine

El valor inicial indica que no ha sido modificado, una vez se modifique el control se remueve la clase ng-pristine y se agrega la clase ng-dirty, por lo tanto una clase ng-dirty solo significa que el campo ha sido modificado

Una vez modificado el control, no volverá a ser pristine

Go!
